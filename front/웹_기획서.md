# 동물병원 생체 모니터링 시스템 웹 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트 목적
동물병원에서 입원한 환자(반려동물)의 생체 신호(심박수, 산포도, 온도, IR 데이터)를 실시간으로 모니터링하고 관리하는 웹 애플리케이션입니다.

### 1.2 주요 기능
- 허브 및 디바이스 등록 및 관리
- 환자 정보 등록 및 관리
- 실시간 생체 신호 모니터링
- 과거 기록 조회 및 다운로드
- 병원 계정 관리

### 1.3 기술 스택
- React 19.2.0
- React Router DOM 7.9.6
- Vite 7.2.2
- CSS (커스텀 스타일링)

---

## 2. 전체 페이지 구조 및 네비게이션

### 2.1 페이지 목록
1. **로그인 페이지** (`/login`)
2. **회원가입 페이지** (`/register`)
3. **대시보드 페이지** (`/dashboard`) - 메인 페이지
4. **하드웨어 관리 페이지** (`/hardware`) - 탭: 허브 관리, 디바이스 관리
5. **환자 관리 페이지** (`/patients`) - 탭: 전체, 입원중, 퇴원
6. **기록 관리 페이지** (`/records`) - 필터: 날짜, 환자, 디바이스
7. **모니터링 페이지** (`/monitoring/:patientId`) - 동적 라우팅
8. **내정보 페이지** (`/profile`)
9. **사용 가이드 페이지** (`/guide`) - 새 창으로 열림

### 2.2 네비게이션 구조
```
Header (고정 상단)
├── 로고 (좌측)
├── 네비게이션 메뉴 (중앙)
│   ├── 대시보드
│   ├── 하드웨어 관리
│   ├── 환자 관리
│   └── 기록 관리
└── 사용자 영역 (우측)
    ├── 병원명 표시
    ├── 내정보 링크
    └── 로그아웃 버튼
```

## 3. 페이지별 상세 사양

### 3.1 로그인 페이지 (`/login`)

#### 레이아웃
- **전체 배경**: 단색 배경 (#f5f5f5 또는 유사)
- **중앙 정렬**: 수직/수평 중앙 정렬
- **컨테이너**: 흰색 카드 형태, 둥근 모서리, 그림자 효과

#### 구성 요소
1. **로고/타이틀 영역**
   - 제목: "생체 모니터링 시스템"
   - 폰트 크기: 대형 (1.5rem 이상)
   - 폰트 굵기: Bold
   - 색상: #2c3e50 또는 진한 회색

2. **로그인 폼**
   - 이메일 입력 필드
     - 라벨: "이메일"
     - 타입: email
     - 플레이스홀더: "이메일을 입력하세요"
     - 필수 입력
   - 비밀번호 입력 필드
     - 라벨: "비밀번호"
     - 타입: password
     - 플레이스홀더: "비밀번호를 입력하세요"
     - 필수 입력
   - 로그인 버튼
     - 배경색: #3498db (파란색)
     - 텍스트: "로그인"
     - 전체 너비
     - 호버 효과

3. **회원가입 링크**
   - 하단 중앙 배치
   - 텍스트: "계정이 없으신가요? 회원가입"
   - 링크 색상: #3498db

#### 스타일 가이드
- 컨테이너 최대 너비: 400px
- 패딩: 2rem
- 입력 필드 높이: 40-45px
- 버튼 높이: 45-50px
- 필드 간 간격: 1rem

   - 섹션 제목: "계정 정보"

   - 병원 이메일 입력 필드
   - 병원 전화번호 입력 필드

4. **마케팅 수신 동의 섹션**
   - 체크박스
   - 라벨: "마케팅 정보 수신에 동의합니다"

5. **회원가입 버튼**

   - 파란색 배경

6. **로그인 링크**
   - 하단: "이미 계정이 있으신가요? 로그인"

#### 스타일 가이드
- 섹션 간 간격: 2rem

## 10. 특수 기능 상세

### 10.1 허브 등록 플로우
1. "허브 등록" 버튼 클릭
2. 모달 열림
3. "USB 선으로 허브를 연결하세요" 안내 표시
4. "연결된 디바이스 검색" 버튼 클릭
5. 검색 중 상태 표시
6. 연결 가능한 디바이스 목록 표시
7. 디바이스 선택 (클릭)
8. WiFi ID, 비밀번호 입력
9. "등록" 버튼 클릭
10. 등록 완료 알림

### 10.2 디바이스 등록 플로우
1. "디바이스 등록" 버튼 클릭
2. 모달 열림
3. "허브를 켜주세요" 안내
4. "허브를 디바이스 등록 모드로 전환" 버튼 클릭
5. "연결하고자 하는 모든 디바이스를 켜주세요" 안내
6. "디바이스 검색" 버튼 클릭
7. 스캔된 디바이스 목록 표시
8. 각 디바이스에 대해:
   - "LED 깜빡이기" 버튼으로 식별
   - "등록하기" 버튼 클릭
   - 디바이스명 입력
   - 취소 또는 계속
9. 모든 디바이스명 입력 후 "최종등록하기" 버튼 클릭
10. 등록 완료 알림

### 10.3 모니터링 페이지 플로우
1. 대시보드에서 "모니터링하기" 버튼 클릭
2. 모니터링 페이지로 이동
3. 환자 정보 및 현재값 표시
4. 차트 탭 선택 (IR, 심박수, 산포도, 온도)
5. 실시간 차트 확인
6. "더보기" 버튼으로 환자 상세 정보 확인
7. "대시보드로 돌아가기" 버튼으로 복귀
